Good question. Here's an honest breakdown of free-tier options that can actually run this stack, and a recommendation.

---

## The honest constraint

This backend needs: PostgreSQL, Redis, MinIO (S3-compatible storage), MLflow, Celery workers, and the FastAPI app itself — plus optionally Kafka. That's 5–6 services. **No single free tier runs all of them together.** The strategy is to use one free platform for the app + workers, and free-tier managed services for the databases.

---

## Best free-tier combination

**Recommended stack:**

| Service | Provider | Free tier |
|---|---|---|
| FastAPI app | **Railway** | $5 free credit/month (enough for dev) |
| Celery worker | Railway | Same project, second service |
| PostgreSQL | **Neon.tech** | 512 MB, always free |
| Redis | **Upstash** | 10K commands/day, always free |
| Object storage (MinIO/S3) | **Cloudflare R2** | 10 GB free, no egress fees |
| MLflow tracking | Railway | Third service in same project |
| Kafka / event bus | Set `EVENT_BACKEND=noop` | Skip for now, enable later |

This gets you a fully running backend at zero cost for development and demos.

---

## Step-by-step setup

### Step 1 — PostgreSQL on Neon

1. Go to **neon.tech**, create a free account
2. Create a new project → copy the connection string, it looks like:
```
postgresql://user:password@ep-xxx.us-east-2.aws.neon.tech/neondb?sslmode=require
```
3. Change `postgresql://` to `postgresql+asyncpg://` — that's your `DATABASE_URL`

---

### Step 2 — Redis on Upstash

1. Go to **upstash.com**, create a free account
2. Create a Redis database → choose the region closest to your Railway deployment
3. Copy the `REDIS_URL` — it looks like:
```
rediss://default:password@xxx.upstash.io:6379
```

---

### Step 3 — Object storage on Cloudflare R2

1. Go to **cloudflare.com**, create a free account
2. Go to R2 → Create bucket → name it `research-artifacts`
3. Create a second bucket named `feature-store`
4. Go to **Manage R2 API Tokens** → Create token with read+write permissions
5. Note your:
   - Account ID (from the R2 dashboard URL)
   - Access Key ID
   - Secret Access Key
   - Endpoint: `https://<account_id>.r2.cloudflarestorage.com`

---

### Step 4 — Deploy to Railway

1. Go to **railway.app**, sign up with GitHub
2. Click **New Project → Deploy from GitHub repo** → connect your backend repo

3. In your repo root, create this `Dockerfile`:

```dockerfile
FROM python:3.12-slim

WORKDIR /app

# System deps for psycopg2, torch, etc.
RUN apt-get update && apt-get install -y \
    gcc g++ git curl \
    && rm -rf /var/lib/apt/lists/*

COPY backend/requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

COPY backend/ .

EXPOSE 8000

CMD ["uvicorn", "app.main:app", "--host", "0.0.0.0", "--port", "8000"]
```

4. Also create `Dockerfile.worker` for the Celery worker:

```dockerfile
FROM python:3.12-slim

WORKDIR /app

RUN apt-get update && apt-get install -y gcc g++ git \
    && rm -rf /var/lib/apt/lists/*

COPY backend/requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

COPY backend/ .

CMD ["celery", "-A", "app.workers.celery_app", "worker", "--loglevel=info", "--concurrency=2"]
```

5. In Railway: create **two services** in the same project — one for the API (uses `Dockerfile`) and one for the worker (uses `Dockerfile.worker`).

---

### Step 5 — Set environment variables in Railway

In each Railway service → **Variables** tab, add:

```env
# Database
DATABASE_URL=postgresql+asyncpg://user:password@ep-xxx.neon.tech/neondb?ssl=require

# Redis
REDIS_URL=rediss://default:password@xxx.upstash.io:6379
CELERY_BROKER_URL=rediss://default:password@xxx.upstash.io:6379/1
CELERY_RESULT_BACKEND=rediss://default:password@xxx.upstash.io:6379/2

# Cloudflare R2 (S3-compatible)
S3_ENDPOINT_URL=https://<account_id>.r2.cloudflarestorage.com
S3_ACCESS_KEY=your_r2_access_key
S3_SECRET_KEY=your_r2_secret_key
S3_BUCKET_ARTIFACTS=research-artifacts
S3_BUCKET_FEATURES=feature-store

# MLflow (point to itself or skip for now)
MLFLOW_TRACKING_URI=http://localhost:5000

# Event bus — disable Kafka for now
EVENT_BACKEND=noop

# App
APP_ENV=production
SECRET_KEY=change-this-to-a-random-string
CORS_ORIGINS=["https://your-frontend-url.vercel.app","http://localhost:5173"]

# Market Data Layer (your external data source URL, or a mock)
MARKET_DATA_URL=https://your-market-data-service.com
```

---

### Step 6 — Run database migrations

In Railway → your API service → **Shell** tab (or use the Railway CLI):

```bash
alembic upgrade head
```

This creates all the tables. Run this once after first deploy and after any schema changes.

---

### Step 7 — Point your frontend at Railway

In your frontend `.env.production`:
```
VITE_API_BASE_URL=https://your-api-service.up.railway.app/api/v1
```

Redeploy the frontend (Vercel/Netlify picks this up automatically).

## What won't work on free tier

| Feature | Reason | Workaround |
|---|---|---|
| FinBERT / LSTM training | Needs GPU or large RAM | Use XGBoost / LightGBM only |
| Kafka event bus | Not free-tier friendly | Set `EVENT_BACKEND=noop` |
| MLflow UI | Needs a persistent server | Use Railway's paid tier ($5/mo) or skip |
| tsfresh comprehensive | Very slow on 512MB RAM | Use `feature_set=minimal` |
